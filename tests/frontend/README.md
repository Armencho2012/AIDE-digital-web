# Frontend Test Plan

The frontend is TypeScript React running on Vite. This scaffold intentionally does not add Python frontend tests.

Recommended stack:

- Unit/component: Vitest plus React Testing Library.
- Route and user-flow E2E: Playwright or Cypress.
- Accessibility: axe integration in component and E2E tests.
- Network isolation: MSW or framework-native request interception for Supabase Edge Function calls.

Priority flows:

- Auth page validation, signup, signin, password reset, and redirect handling.
- Dashboard analysis submission, media attachment, usage-limit lock, and saved content creation.
- Library/detail pages for quiz, flashcards, course, podcast generation, PDF export, and missing asset regeneration.
- Chat page streaming response rendering and empty/error response states.
- Billing checkout path for direct Gumroad URLs and Edge Function fallback.
- Knowledge map edit, save, export, outline generation, clipboard copy, and scan-for-gaps flow.

Frontend E2E is flag-only in `tests/TEST_PLAN.md`; no browser tests were executed or generated by the agent.
